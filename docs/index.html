Abstract —

Bioinformatics Agent Studio couples established statistical pipelines with constrained language-model agents. The computation — differential expression, classification, pathway enrichment — is performed by domain-standard tools. The interpretation — quality-control review, results narrative, literature synthesis — is drafted by agents that receive explicit structured input and operate under strict scientific constraints. The result is a workflow that produces a publication-style report from a raw count matrix in a single command, while preserving human review at every decision point. The system is fully containerized and generic: it accepts arbitrary RNA-seq datasets, not a fixed example.

Status. RNA-seq pipeline production-ready — generic, containerized, tested on independent data.

Scope. Five agents across three analysis domains.

Runtime. ~80 s end-to-end on an 8-sample study.
§ 1

The system

Two layers. A statistical layer (DESeq2, scikit-learn, g:Profiler) does the science. An agent layer (Claude Sonnet) converts the numerical output into written scientific text. A command-line orchestrator runs the sequence and logs every step.

Agents never see raw data and never make autonomous calls. Each reads a complete JSON record and writes one kind of output — a deliberate constraint, not a limitation.

counts.csv + metadata.csv → DESeq2 → enrichment → QC agent → report agent → report.html

Five specialists. Each takes structured JSON, follows a fixed scientific brief, and emits a single document type.

§ 4

Design principles

i

Constrained agents

Agents are specialists with fixed inputs and outputs — not autonomous decision-makers. The science is done by validated tools.

ii

Explicit input

Every agent receives a complete JSON record. No agent recomputes a value from aggregate statistics.

iii

Human in the loop

Generated narrative is a first draft. Scientific conclusions require human validation before delivery.

§ 5

Implementation

Reproducible by construction — the entire environment is pinned in a Docker image, so a run on any machine yields identical output.
